# 竞品情报报告 Skill

为手机厂商游戏分发业务生成战略级竞品情报报告。分析视角始终锚定：**我们是手机厂商的游戏分发业务，核心价值是系统级入口+分发+增值服务，不是内容社区也不是游戏厂商。**

## 第一步：判断报告类型

收到任务后，先判断要生成哪种报告：

**A · 周报**（默认，每周一）
- 触发词：竞品周报、本周竞品、weekly、定时任务
- 覆盖本周所有监控范围的新动态
- 每条动态300-500字三层结构

**B · 专题报告**（单一竞品深度）
- 触发词：专题、深度分析、[具体竞品名]
- 单一竞品完整拆解，六节固定结构
- 信息层全部附可点击来源链接

判断后直接执行，不需要询问确认。

---

## 第二步：信息采集

**采集方式按当前环境的能力自动选择：**

- **环境具备联网搜索能力时**（任意可用的网页搜索工具，如 WebSearch / web_search 等）：按下方搜索词模板主动采集。
- **环境不具备联网搜索能力时**：不要编造数据。先告知用户"当前环境无联网搜索，无法自动采集竞品动态"，请用户**粘贴素材或提供来源 URL**（新闻链接、报告片段、官方公告等）。拿到素材后，skill 退化为"分析 + 排版 + 自检"模式，仍可正常交付报告——只是信息层数据由用户提供而非自动抓取。

无论哪种方式，**无来源URL的数据一律不得写入报告**。

### 搜索词模板（仅在具备联网搜索能力时使用）

**第一类竞品（每周必搜）：**
```
抖音小游戏 [当前年月]
快手小游戏 游戏分发 [当前年月]
微信小游戏 新游 政策 [当前年月]
TapTap 新功能 [当前年月]
```

**第二类竞品（每周必搜）：**
```
小红书 游戏 AI [当前年月]
豆包 游戏 [当前年月]
B站 游戏 AI [当前年月]
```

**第三类竞品（扫描，有亮点才写）：**
```
Google Play AI推荐 [当前年月]
好游快爆 新功能 [当前年月]
AI游戏伴游 新产品 [当前年月]
```

**AI行业信号：**
```
AI游戏 大模型 [当前年月]
游戏推荐算法 AI [当前年月]
```

### 来源优先级（高→低）
1. `[官方]` 平台官方博客/发布会报道
2. `[媒体]` 36氪/GameLook/葡萄书/17173/新浪财经
3. `[机构]` QuestMobile/极光月狐/DataEye
4. `[解读]` 知乎/虎嗅等分析文章（需额外说明可信度）
5. **禁止使用**：无日期内容、匿名帖子、无来源数据

### 时效要求
- 周报：只收录**14天内**的动态
- 专题：核心数据允许使用3个月内报告，需标注采集时间

## 第三步：分析框架

**先判断竞品类别，再用对应框架分析。**

### 竞品三分类

| 类别 | 代表产品 | 分析视角 |
|------|---------|---------|
| 第一类·流量竞争 | 抖音、快手、微信小游戏、TapTap、官渠 | 抢的是哪个场景的分发流量？ |
| 第二类·决策替代 | 小红书、豆包、B站、抖音搜索 | 在游戏决策链路哪个节点切入？ |
| 第三类·学习参考 | Google Play、Steam、好游快爆、华为/小米游戏中心 | 有哪些值得我们移植的亮点？ |

### 第一类分析框架
```
1. 是什么？（功能/政策/数据 + 来源URL + 日期）
2. 抢的是哪个场景？（发现/下载/付费/留存）
3. 规模信号：有数据佐证吗？
4. 我们对应场景的现状？
5. 判断：防守/跟进/差异化，优先级高/中/低
```

### 第二类分析框架
```
1. 是什么？（功能/体验 + 来源URL + 日期）
2. 在游戏决策链路哪个节点切入？（找游戏/比较/攻略/付费决策）
3. 用户路径：用完之后去哪？
4. 与我们在建能力的重叠度（game_insight/user_insight/搜索意图）
5. 判断：对我们决策入口价值的侵蚀程度，建议动作
```

### 第三类分析框架
```
1. 是什么？（功能/设计 + 来源URL + 日期）
2. 它解决了什么用户问题？用了什么方法？
3. 哪个设计值得移植？对应我们哪个在建模块？
4. 执行前提：依赖我方哪些已有/未有的能力或供给？
5. 建议：直接参考/改造后用/仅作信号，优先级
```

> 详细分析视角和判断底座见 `references/strategy-base.md`

## 第四步：生成报告

### 周报结构
```
核心判断（本周最高优先级信号，1-2条）
第一类动态（3-5条，每条三层结构）
第二类动态（2-3条，每条三层结构）
第三类动态（1-2条，有亮点才写）
AI行业信号（1-2条简报）
本周总结（结论≤5条，每条必须有明确动作建议）
```

### 专题报告结构（固定六节）
```
核心判断（结论前置）
01 · 产品概览（定位+规模数据+现阶段定性）
02 · 用户与场景（画像+典型路径+关键数据）
03 · 产品能力拆解（核心功能+差异化机制+已知短板）
04 · 商业结构（变现路径+对厂商价值主张）
05 · 综合评估（星级评估表+优势+风险）
06A · 威胁视角（路径+升级信号+我们的弱势+建议动作）
06B · 学习视角（可移植机制，每条必须含执行前提）
行动排序（P0/P1/P2优先级矩阵）
信息盲区（外部/内部/行为层面三类）
来源说明（全部可点击链接）
```

### 输出格式要求
- **格式**：HTML，侧边栏导航，来源标签可点击跳转
- **视觉规范**：见 `references/html-template.md`（DM Sans字体、白色卡片、蓝色accent、来源badge体系）
- **文件命名**：
  - 周报：`竞品周报_W[周次]_[年份].html`
  - 专题：`专题报告_[竞品名]_[年份]W[周次].html`

## 第五步：交付前自检（必须过，不得跳过）

### A · 结构完整性
- [ ] 每条核心判断能在正文找到支撑段落？若无 → 降级为「待验证假设」
- [ ] 所有推断已标注 `[推断]`？
- [ ] 信息盲区包含三类：①外部数据缺失 ②我方内部数据缺失 ③行为层面缺失
- [ ] 所有建议动作已按「紧迫性×执行能力」排序？

### B · 推理链完整性
每条借鉴建议确认三项：
- [ ] 触发条件说明了吗？
- [ ] 执行前提（依赖我方哪些能力/供给/数据）说明了吗？
- [ ] 接驳点映射到我方具体模块了吗？（不能只说"有启发"）

### C · 威胁判断质量
- [ ] 考虑了"同时分发"的可能性？（竞品选择≠放弃我们）
- [ ] 时长/注意力论断基于行为习惯变化而非零和假设？
- [ ] 威胁分级区分了「已发生」vs「趋势信号」vs「待验证假设」？

### D · 信息层质量（本skill新增）
- [ ] 所有数据附有来源URL？
- [ ] 所有来源确认了发布日期？
- [ ] 周报动态全部在14天内？
- [ ] 官方数据 vs 媒体转述 vs 推断三者已显式区分？
- [ ] 判断层无模糊语言（"可能""也许""值得关注"不算判断）？

---

## 质量红线（任何情况下不得违反）

1. 数据无来源URL → 不得写入报告
2. 来源无法确认日期 → 不得使用
3. 核心判断无正文支撑 → 必须降级为「待验证假设」
4. 判断层出现模糊语言 → 必须重写
5. 借鉴建议无「执行前提」 → 不得交付
6. 数字是否准确无法核实 → 标注存疑或删除，不得按事实陈述
