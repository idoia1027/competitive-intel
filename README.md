# competitive-intel

把游戏分发竞品的公开动态，转成有来源、有分析、有决策动作的情报报告；在合适的时候，再导出一份侧边栏可导航的 HTML 交付物。

`competitive-intel` `游戏分发` `竞品分析` `战略情报` `定期报告`

## 安装

```
npx skills add https://github.com/idoia1027/competitive-intel
```

安装后，下面这类请求会优先触发这个 skill：

- 出竞品周报
- 分析抖音小游戏
- 竞品专题报告：TapTap
- 本周有什么新竞品动态
- scheduled竞品任务

## 背景

竞品分析常见的三个问题：

- **信息层停留**：只记录"发生了什么"，缺少为什么重要、对我们意味着什么
- **判断模糊**：充斥"可能""也许""值得关注"等无法行动的语言
- **借鉴建议落不了地**：没有说清执行前提（依赖哪些我方能力 / 供给 / 数据）

`competitive-intel` 强制三层结构（信息层 / 分析层 / 判断层），以手机厂商游戏分发业务为视角底座，对竞品分三类（流量竞争 / 决策替代 / 学习参考）分别走对应分析框架，交付前必须过四类自检。所有数据来源可溯源，所有借鉴建议必须标注执行前提。

### 两种报告形态

| 形态 | 触发词 | 周期 |
|------|--------|------|
| 竞品周报 | "出竞品周报"、"本周竞品"、定时任务 | 每周一 |
| 竞品专题 | "分析[竞品名]"、"专题报告"、"深度分析" | 每两周一篇 |

### 分析视角

始终锚定：**我们是手机厂商的游戏分发业务，核心价值是系统级入口 + 分发 + 增值服务，不是内容社区也不是游戏厂商。**

竞品按威胁类型分三类：

| 类别 | 代表产品 | 分析视角 |
|------|---------|---------|
| 第一类 · 流量竞争 | 抖音、快手、微信小游戏、TapTap | 抢的是哪个场景的分发流量？ |
| 第二类 · 决策替代 | 小红书、豆包、B站、抖音搜索 | 在游戏决策链路哪个节点切入？ |
| 第三类 · 学习参考 | Google Play、Steam、好游快爆、华为/小米游戏中心 | 有哪些值得移植的亮点？ |
